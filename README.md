# 财务中心后台静态演示版

本目录是基于 `docs/finance-prd.md` 优化后的财务中心运营后台原型。它从“按表平铺的研发演示”调整为“按四大业务模块组织的正式演示版财务中心”，用于产品评审、运营走查、财务演示和研发理解页面边界。

## 打开方式

直接用浏览器打开：

```text
/Users/sikon/Downloads/ruoyi-finance/docs/admin-demo/index.html
```

无需启动后端服务，也无需安装依赖。

## 演示定位

- 纯 HTML/CSS/Vanilla JS。
- 所有数据均为 `app.js` 内的本地 Mock。
- 不调用真实 API，不连接数据库，不修改后端 Java 或 SQL。
- 新增、编辑、删除只影响浏览器内存，刷新页面后恢复初始 Mock 数据。
- “往来台账”是产品补充原型，当前后端代码和 SQL 未实现真实表、审批流、借款接口或还款核销接口。
- “月结中心”和“站点资金池”是产品补充原型，当前后端代码和 SQL 未实现真实月结表、资金池表、扣款接口或额度调整接口。

## 四大模块结构

- 工作台：财务总览、待办中心、异常预警、流程看板、操作审计，作为支撑入口。
- 出入款帐户：出入款帐户总览、私有账户、站点资金池、月结总览、月结试算、月结账单、资金池流水、额度调整。
- 收银流水：收银流水总览、收银申请、收银流水、收银历史、充值/提现场景、收银策略、交易字典、风控规则、黑/红名单、风险命中。
- 记账模板：记账模板总览、记账模板、记账策略、记账凭证、试算平衡、记账历史、记账分类。
- 客户余额：客户余额总览、账户总览、余额明细、账户开通、账户类型、账户对象、站点身份、往来总览、站点借款、代理欠款、代理借款、还款核销。
- 研发参考：表关系、状态字典、代码风险、跨模块策略总览，作为程序参考入口。

四大模块整合的是运营导航和使用路径，不代表后端 Controller、权限点、数据表或责任人合并。独立页面仍保留各自路径、权限和功能说明。

## 页面能力

- 每页包含管理目标、核心口径、责任边界、业务摘要卡片、运营报表、处理优先级、快捷入口、筛选区、精简明细表格和功能说明。
- 四大模块均有只读总览页，用于统一展示同类信息、快捷跳转和模块边界。
- 详情弹窗先展示业务结论、状态、金额和责任人，再按用户信息、交易信息、金额信息、状态信息、记账信息、技术信息分组展示完整字段。
- 列表默认只展示高频运营字段，低频技术字段和完整字段放入详情弹窗，减少页面杂乱。
- 运营报表区按当前筛选结果展示状态分布、金额与规模、处理优先级，模拟正式后台常见报表视角。
- 功能说明弹窗包含页面定位、运营场景、核心操作、字段分组、状态流转、关联模块、后端路径、关联表和实现注意事项。
- 技术字段不作为运营页面主信息，但可在详情和说明中找到。
- 顶部支持角色切换、时间范围切换、通知入口、全局搜索下拉和重置演示数据。
- 工作台新增正式驾驶舱图表：收支趋势、收支分类、资金流水瀑布、12 个月资金热力图和异常预警。
- 收银申请页为存款/取款待审申请增加“通过/驳回”按钮和二次确认弹窗；操作只更新本地 Mock 状态，不调用真实审批接口。
- 退款管理页演示退款申请、审核、退款处理中、已退款/已拒绝状态流。
- 渠道对账页演示系统流水、渠道流水、差异金额、差异原因和处理路径。
- 报表中心和自定义报表页演示日报、月报、损益报表、账龄分析、现金流预测、字段选择、聚合口径、预览和导出提示。
- 顶部角色切换仍用于演示不同岗位的菜单和按钮差异，但不再提供独立权限矩阵页面。
- 月结账单页可演示“确认扣款/退回调整”；确认扣款只扣站点资金池，资金不足会阻断并标记异常，不允许扣成负数。
- 额度调整页可演示总站增加/减少站点资金池额度，并追加资金池流水；刷新后恢复初始 Mock。
- 往来台账页面采用“汇总 + 明细”展示：先看站点/代理应收、应付、已还、未结清和逾期，再进入具体台账和还款核销。

## 页面类型与合并规则

- 管理页：对应独立路径和权限，可演示查询、新增、编辑、删除、导出。
- 看板/流程/检查页：只做总览、排障、流程说明和跳转，不展示新增、编辑、删除。
- 研发参考页：只读展示表关系、状态字典、代码风险和跨模块策略总览。
- 独立 Controller、独立权限或独立责任人的模块不可合并编辑；收银策略和记账策略已拆为独立管理页。

## 月结中心规则

- 总站每月对下属站点做月结，月结结果扣减站点资金池。
- 站点资金池是单个站点可提现金额，不等同于会员或代理个人余额。
- 资金池增加来源：站点充值、代理充值、会员充值、总站增加额度。
- 资金池减少来源：站点提现、代理提现、会员提现、总站减少额度、月结扣款。
- 月结金额展示“系统试算金额 + 人工调整金额 = 最终应扣金额”。
- 资金池不足时阻断扣款，月结单标记资金不足并进入异常预警，不自动形成负数或往来欠款。

## 往来台账规则

- 站点可以借款给代理，站点视角为借款/应收，代理视角为欠款/应付。
- 代理可以欠上级代理或站点，也可以借款给下级代理。
- 会员/用户不会产生借款、欠款或还款核销台账。
- 当前页面只做原型展示，真实接入需新增台账表、审批流、额度控制、还款凭证、核销和操作审计。

## 建议真实接口接入顺序

1. 先接总览、待办、异常预警的聚合数据。
2. 再接收银流水域的收银申请、收银流水、收银历史和收银策略。
3. 然后接出入款帐户域的站点资金池、月结试算、月结账单、资金池流水和额度调整。
4. 再接客户余额域的账户总览、余额明细、账户开通和往来台账。
5. 再接记账模板域的记账模板、记账策略、记账凭证和试算平衡。
6. 最后补齐操作审计、历史归档、研发参考类页面。
